new Snapshot logic:

ok, from my understanding the obsolete collections should be skipped, but loading an obsolete collection is really complicated whit the current system and obsolete collections.

For this reason I propose a completly new system.

Whenever a snapshot is created you add them to a list of all snapshots. When a snapshot is you add them to the list as well. Also you go back the list and add them to the modified_snapshot list until you hit the same number of the restored snapshot again, also add that one to the modified_snapshot list. (make sure you just use the snapshto numbers for that)

then you create a modified_collections list from all unique collections in the modified_snapshot list. (we need a function to retrieve a list of snapshots based on a list of numbers)

Then you go back from the back of the modified_snapshot list and collect all snapshots into a collected_snapshots list and keep track of all collected_collections until the collected_collections list matches the modified_collections list. Dont add snapshots to the collected_snapshots list, that only contain collections already in collected_colllections list.

Now we create a restore point of all collected_collections list, this one should also be added to the overall snapshot list.

now we restore the collected_collections list from the back, meaning the ones which were added last. (as a reference, the loaded snapshot always has to be restored last).

great, and in the end we modify collections obsolete status, based on the modified snapshot list: 

from the right of the list mark all snapshots as not obsolete until the next number in the list is larger.
and in general mark all numbers larger then the loaded snapshot number as obsolete.
